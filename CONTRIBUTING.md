> **Join the Community**

We welcome contributions from everyone! Whether you're fixing a bug, adding a feature, or improving documentation, your help is appreciated.

---

## 1. Code of Conduct

-   **Be Respectful**: Treat everyone with kindness and respect.
-   **Be Constructive**: Feedback should be helpful, not hurtful.
-   **Be Inclusive**: We welcome people of all backgrounds and skill levels.

---

## 2. How to Contribute

### 2.1. Reporting Bugs
1.  Check the [Issues](https://github.com/Palolo875/kensho-1/issues) tab to see if it's already reported.
2.  Open a new Issue with a clear title and description.
3.  Include steps to reproduce the bug.

### 2.2. Suggesting Features
1.  Open a new Issue with the label `enhancement`.
2.  Describe the feature and why it would be useful.

### 2.3. Submitting Code
1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  **Create a Branch**: `git checkout -b feature/my-new-feature`.
4.  **Make Changes**: Write code and tests.
5.  **Commit**: Use conventional commits (e.g., `feat: add new weather agent`).
6.  **Push**: `git push origin feature/my-new-feature`.
7.  **Open a Pull Request**: Describe your changes and link to any relevant issues.

---

## 3. Development Standards

### 3.1. Code Style
-   We use **ESLint** and **Prettier**.
-   Run `npm run lint` before committing.

### 3.2. Testing
-   All new features must have **Unit Tests**.
-   Run `npm run test:unit` to verify.

### 3.3. Documentation
-   Update `COMPONENTS.md` if you change an API.
-   Add comments to complex code blocks.
